# Catch the Liar — Hallucination Detection in AI-Generated Text

## Technical Report

---

## 1. Problem Statement

### 1.1 Background

Large Language Models (LLMs) are increasingly deployed to summarize scientific content. However, these models can be **manipulated** — causing them to hallucinate, drift off-topic, introduce factual errors, or produce outputs subtly inconsistent with the source material. This challenge, part of the SCSE AI Hackathon, tasks participants with identifying which of two AI-generated summaries of the same scientific article originates from a **manipulated (poisoned) model**.

### 1.2 Task Definition

Given a pair of summaries (Text A, Text B) of the same source article:

- **Output = 1** → Text A is from the manipulated model (File 1 is the "liar")
- **Output = 2** → Text B is from the manipulated model (File 2 is the "liar")

The other text in each pair is the clean, faithful summary.

### 1.3 Constraints

| Constraint | Detail |
|------------|--------|
| **No pretrained models** | No BERT, RoBERTa, GPT, Word2Vec, GloVe, FastText, or any pretrained weights |
| **From-scratch only** | All representations must be learned from scratch on provided/public data |
| **Metric** | Accuracy on a held-out test set |
| **Submission** | CSV with id and predicted label (1 or 2) |

### 1.4 Key Challenges

| Challenge | Detail |
|-----------|--------|
| **Extreme data scarcity** | Only 95 labeled training pairs |
| **No pretrained representations** | Must build all feature extractors from scratch |
| **Subtle hallucinations** | Manipulated text is LLM-generated — fluent and plausible |
| **Large test set** | 1068 unlabeled test pairs (11× training size) |
| **Domain-specific** | Texts are astronomy/astrophysics summaries |

## 2. Datasets

### 2.1 Competition Dataset

| Split | Pairs | Source |
|-------|-------|--------|
| Train | 95 | ESO "The Messenger" journal articles |
| Test | 1068 | Same source, labels withheld |

Each pair consists of two text files (`file_1.txt`, `file_2.txt`) — both are LLM-generated summaries of the same source article. One was generated by a **clean model**, the other by a **manipulated (poisoned) model**. The label indicates which file contains the clean (real) output.

**Label distribution (train):** 46 pairs with label=1 (file_1 is real), 49 pairs with label=2 (file_2 is real) — approximately balanced.

**Text characteristics:**
- Average text length: ~200–400 words per summary
- Domain: Astronomy, astrophysics, space instrumentation
- Language: English, scientific register

**Example of hallucination observed in training data:**
- Real summary: discusses VISTA/ESO telescope surveys, mentions specific instruments (VIRCAM), surveys (VVV), and quantitative measurements
- Fake summary: fabricates entirely different content — invents a "China relay network", uses exclamation marks, vague language, and informal tone

### 2.2 External Dataset: arXiv astro-ph Abstracts

To compensate for the tiny training set, we incorporated the publicly available [Cornell arXiv metadata dataset](https://www.kaggle.com/datasets/Cornell-University/arxiv) (`arxiv-metadata-oai-snapshot.json`, ~4 GB). We extracted **10,000 abstracts** from the `astro-ph` (astrophysics) category.

**Purpose:** Training language models and word embeddings on a large in-domain corpus, providing a statistical baseline of "what real astrophysics text looks like" without seeing any test data.

**Filtering criteria:**
```python
if 'astro-ph' in paper.get('categories', '') and paper.get('abstract', '').strip():
    corpus.append(paper['abstract'])
```

**Why arXiv astro-ph?** The competition texts are scientific astronomy summaries. By training on arXiv astrophysics abstracts, our language models learn the vocabulary, style, and statistical patterns of genuine scientific writing in this domain — making them sensitive to deviations introduced by manipulation.

## 3. Approach Overview

Our approach is a **feature-engineered stacking ensemble** with **iterative pseudo-labeling**. The pipeline has six stages:

```
┌─────────────┐    ┌──────────────┐    ┌────────────────┐
│ 1. Language  │───▶│ 2. Feature   │───▶│ 3. Feature     │
│    Models    │    │    Extraction │    │    Selection    │
└─────────────┘    └──────────────┘    └────────────────┘
                                              │
                                              ▼
┌─────────────┐    ┌──────────────┐    ┌────────────────┐
│ 6. Final    │◀───│ 5. Pseudo-   │◀───│ 4. Stacking    │
│    Predict  │    │    Labeling   │    │    Ensemble     │
└─────────────┘    └──────────────┘    └────────────────┘
```

### 3.1 Design Philosophy

1. **No pretrained models** — Every representation (embeddings, language models, TF-IDF) is trained from scratch on arXiv + competition data
2. **Feature engineering over deep learning** — With 95 training samples, neural networks would overfit catastrophically; hand-crafted features with tree-based ensembles are far more data-efficient
3. **Exploit the test set** — With 1068 unlabeled test pairs (11× training), pseudo-labeling provides massive additional training signal
4. **Target hallucination signatures** — Rather than generic NLP features, we designed features that specifically detect the known failure modes of manipulated LLMs

## 4. Language Models (From Scratch)

We train four language models from scratch to capture the statistical properties of real vs. manipulated text.

### 4.1 Multi-Order Character N-gram Language Model

A character-level language model estimates the probability of a character given its preceding context. We train an **ensemble of three character n-gram LMs** with orders n ∈ {3, 4, 5}.

**Model definition:** For a text T with characters c₁, c₂, …, cₘ:

```
P(cᵢ | cᵢ₋ₙ₊₁ … cᵢ₋₁) = [count(cᵢ₋ₙ₊₁ … cᵢ) + k] / [count(cᵢ₋ₙ₊₁ … cᵢ₋₁) + k·|V|]
```

where k = 0.05 is the Laplace smoothing parameter and |V| is the character vocabulary size.

**Perplexity** is used as the scoring metric:

```
PPL(T) = 2^( -1/N · Σ log₂ P(cᵢ | context) )
```

Lower perplexity indicates the text is more consistent with the training distribution.

We train two separate sets of LMs:
- **Real LM**: Trained on arXiv abstracts + real (clean) training texts
- **Fake LM**: Trained on manipulated (fake) training texts only

The **perplexity ratio** PPL_fake / PPL_real serves as a powerful discriminative feature: real text has lower perplexity under the real LM and higher under the fake LM.

**Multi-order ensemble:** The final perplexity is the arithmetic mean across orders 3, 4, and 5, capturing both short-range character patterns (n=3) and medium-range dependencies (n=5).

**Why character-level?** Character LMs capture spelling patterns, morphology, and subword-level style that word-level models miss. Manipulated text often has subtly different character distributions (e.g., more exclamation marks, different punctuation patterns).

### 4.2 Kneser-Ney Interpolated Bigram Language Model

At the word level, we implement an **interpolated Kneser-Ney smoothed bigram LM** — considered one of the best classical smoothing methods.

**Kneser-Ney continuation probability:**

```
P_KN(w) = |{v : count(v, w) > 0}| / |{(u, v) : count(u, v) > 0}|
```

This measures how many unique left contexts word w appears in, rather than its raw frequency. Words that appear in many different contexts (like function words) get higher continuation probability.

**Interpolated bigram probability:**

```
P(w | v) = max(count(v,w) - d, 0) / count(v,·) + λ(v) · P_KN(w)
```

where d = 0.75 is the discount parameter and λ(v) is the interpolation weight ensuring the distribution sums to 1.

**Why Kneser-Ney?** Standard Laplace or Good-Turing smoothing assigns too much probability mass to unseen events. Kneser-Ney's continuation-based lower-order distribution handles unseen bigrams more gracefully, which is critical when scoring text from a domain (astrophysics) where many word combinations are rare.

### 4.3 Leave-One-Out (LOO) Perplexity

To prevent **data leakage** in the training features, we compute perplexities using Leave-One-Out: for each training sample i, we train the LMs on all training texts **except sample i**, then score sample i with these LOO-trained LMs.

```
PPL_LOO(Tᵢ) = PPL_{LM₋ᵢ}(Tᵢ)    where LM₋ᵢ is trained on {Tⱼ : j ≠ i} ∪ arXiv
```

This is computationally expensive (95 × 4 LM fits) but eliminates a significant source of overfitting. Without LOO, the LM has seen the exact text it's scoring, leading to artificially low perplexities that inflate CV scores by ~3–5% while hurting test performance.

## 5. Scratch Word Embeddings (PPMI-SVD)

Since pretrained word embeddings (Word2Vec, GloVe, FastText) are prohibited, we train our own using **Positive Pointwise Mutual Information (PPMI) matrix factorization** — mathematically related to the GloVe objective.

### 5.1 Co-occurrence Matrix Construction

For a vocabulary of |V| = 8,000 words (minimum frequency ≥ 3), we build a weighted co-occurrence matrix C ∈ ℝ^(|V| × |V|) from the arXiv + real text corpus:

```
C_ij = Σ 1/|pos_i - pos_j|    for all (i,j) within window of size 5
```

The inverse-distance weighting gives higher weight to closer word pairs.

### 5.2 PPMI Transformation

The Pointwise Mutual Information between words wᵢ and wⱼ is:

```
PMI(wᵢ, wⱼ) = log[ C_ij · Σ_kl C_kl / (Σ_k C_ik · Σ_k C_kj) ]
```

We apply the positive clamp: PPMI = max(PMI, 0), which discards negative associations that are noisy for rare words.

### 5.3 SVD Factorization

The sparse PPMI matrix is factorized via Truncated SVD:

```
M_PPMI ≈ U · Σ · Vᵀ
```

The word embeddings are the rows of U ∈ ℝ^(|V| × d), L2-normalized. We use d = 80 dimensions.

**Training corpus:** 10,000 arXiv astro-ph abstracts + 95 real training texts.

**Why PPMI-SVD?** Levy & Goldberg (2014) showed that Word2Vec with Skip-gram and Negative Sampling is implicitly factorizing a shifted PPMI matrix. Our explicit PPMI-SVD approach achieves comparable quality without requiring gradient-based training, and is fully from-scratch compliant.

## 6. TF-IDF and Latent Semantic Analysis

### 6.1 Scratch TF-IDF

We implement Term Frequency–Inverse Document Frequency from scratch:

```
TF-IDF(w, d) = (count(w,d) / |d|) · (log[(N+1)/(DF(w)+1)] + 1)
```

where N is the corpus size and DF(w) is the document frequency. Vocabulary is limited to the **top 4,000 words** by raw frequency, filtered to DF > 1.

### 6.2 Latent Semantic Analysis (LSA)

TF-IDF vectors are projected to **60 dimensions** via Truncated SVD, capturing latent semantic topics:

```
TF-IDF vectors (4000-dim) → SVD → 60-dim LSA vectors
```

This provides dense representations for computing document similarities without the curse of dimensionality.

### 6.3 Centroid-Based Features

We compute the **centroid** of all real (clean) training text TF-IDF vectors and use cosine similarity to this centroid as a feature:

```
cos_to_real(T) = cos(TF-IDF(T), mean(TF-IDF(T_real)))
```

Texts closer to the "real centroid" are more likely to be genuine. We apply the same logic to GloVe vectors.

### 7.2 Hallucination-Targeted Features (17 features × 2 texts = 34)

These features directly target the known failure modes of manipulated LLMs. Each was designed after manual inspection of training pairs.

| Feature | Description | Why It Detects Hallucinations |
|---------|-------------|-------------------------------|
| `vagueness` | Density of vague words ("various", "significant", "many", "important") | Manipulated LLMs substitute specifics with vague language |
| `informality_words` | Density of informal words ("awesome", "hopefully", "definitely") | Poisoned models break scientific register |
| `exclamation_count` | Count of "!" in text | Scientific text essentially never uses exclamation marks |
| `question_count` | Count of "?" in text | Unusual in objective summaries |
| `ellipsis_count` | Count of "..." in text | Structural anomaly in scientific writing |
| `informal_punct_ratio` | (exclamations + ellipses) / char_length | Combined informality signal |
| `acronym_density` | Uppercase abbreviations (ESO, VISTA, VVV) per word | Real texts preserve domain-specific acronyms faithfully |
| `exact_number_density` | Decimal numbers (e.g., 1.23) per word | Real texts cite precise measurements from source |
| `unit_number_density` | Numbers with scientific units (3.5 GHz, 100 pc) per word | Scientific measurement specificity |
| `quoted_terms` | Quoted technical terms per word | Domain terminology preservation |
| `parenthetical_defs` | Parenthetical definitions per word | Scientific convention (defining acronyms) |
| `max_word_len` | Longest word in text | Presence of complex technical terms |
| `unique_4gram_ratio` | Unique 4-grams / total 4-grams | Low ratio = repetitive, formulaic text |
| `digit_char_ratio` | Digit characters / total characters | Numeric content density |
| `para_count` | Number of paragraphs | Structural formatting |
| `burstiness` | std(sentence_lengths) / mean(sentence_lengths) | LLM text has unnaturally uniform sentence lengths |
| `zipf_coeff` | Zipf's law slope coefficient | LLM text deviates from natural word frequency distributions |

**Burstiness explained:** Natural human writing has "bursty" sentence lengths — short punchy sentences mixed with long complex ones. LLM-generated text tends to produce sentences of surprisingly uniform length. We measure this as the coefficient of variation of sentence lengths.

**Zipf's Law feature:** Natural language follows Zipf's law, where the frequency of a word is inversely proportional to its rank. We fit a linear regression on log-rank vs. log-frequency:

```
log f(r) = -α · log r + c
```

The slope α (typically ≈ 1 for natural text) deviates systematically in LLM-generated text due to different token sampling distributions.

### 7.5 Pair Feature Construction

For a pair (T₁, T₂) with per-text feature vectors f₁, f₂ ∈ ℝᵈ (where d ≈ 171 per text):

```
x_pair = [ f₁ ‖ f₂ ‖ (f₁ - f₂) ‖ |f₁ - f₂| ‖ f₁ ⊘ (|f₂| + ε) ‖ c ]
```

| Component | Dimensions | Purpose |
|-----------|------------|---------|
| f₁, f₂ | 171 × 2 = 342 | Raw per-text features |
| f₁ - f₂ | 171 | **Signed difference** — captures direction of asymmetry |
| \|f₁ - f₂\| | 171 | **Absolute difference** — captures magnitude regardless of direction |
| f₁ ⊘ (\|f₂\| + ε) | 171 | **Element-wise ratio** — captures relative scale |
| c | ~50 | **Cross-pair features** |
| **Total** | **~905** | |

### 7.6 Cross-Pair Features (~50 features)

These features measure the **relationship between the two texts**, which is critical since both summarize the same source article:

| Feature | Definition | Rationale |
|---------|------------|-----------|
| **Jensen-Shannon Divergence** | JSD(P‖Q) = ½KL(P‖M) + ½KL(Q‖M), M = (P+Q)/2 | Symmetric word distribution distance |
| **Rényi Divergence (α=2)** | D₂(P‖Q) = log₂ Σ P(w)²/Q(w) | Emphasizes tail differences |
| **TF-IDF cosine** | cos(v₁, v₂) | Topic-level similarity |
| **LSA cosine** | cos(LSA(v₁), LSA(v₂)) | Latent topic similarity |
| **GloVe cosine** | cos(g₁, g₂) | Semantic similarity |
| **WMD proxy** | 1 - cos(g₁, g₂) | Word mover's distance approximation |
| **PMI overlap** | Mean PMI of shared vocabulary | Co-occurrence strength |
| **Content word Jaccard** | \|C₁ ∩ C₂\| / \|C₁ ∪ C₂\| for words of length ≥ 5 | **Topic drift detection** |
| **Exclusive content words** | \|C₁ - C₂\| / \|C₁\|, \|C₂ - C₁\| / \|C₂\| | Off-topic fabrication measure |
| **Acronym overlap** | \|A₁ ∩ A₂\| / \|A₁ ∪ A₂\| | Domain entity preservation |
| **Intra-coherence deltas** | coherence(T₁) - coherence(T₂) | Which text jumps topics more |
| **Perplexity deltas** | PPL(T₁) - PPL(T₂) for all LMs | Which text is more surprising |
| **Hallucination feature deltas** | vagueness₁ - vagueness₂, excl_count₁ - excl_count₂, etc. | Direct asymmetry indicators |

## 8. Data Augmentation: Pair Swapping

### 8.1 Method

For each training pair (T₁, T₂) with label y, we construct an augmented sample (T₂, T₁) with flipped label:

```
Original: (T₁, T₂, label=2) → binary y=1
Augmented: (T₂, T₁, label=1) → binary y=0
```

This doubles the training set from **95 → 190 samples** while maintaining perfect label balance (50/50).

### 8.2 Why This Works

The pair feature function φ(T₁, T₂) is **asymmetric** — the delta (f₁ - f₂), ratio (f₁/f₂), and many cross-pair features all change when the pair order is swapped. Specifically:

- δ(T₁, T₂) = f₁ - f₂ = -(f₂ - f₁) = -δ(T₂, T₁)
- r(T₁, T₂) = f₁/f₂ ≠ f₂/f₁ = r(T₂, T₁)

This augmentation therefore creates genuinely new feature vectors, not trivial duplicates.

### 8.3 GroupKFold: Preventing Leakage

To prevent **leakage between augmented pairs** during cross-validation, we use `StratifiedGroupKFold` where each original article pair and its swap share the same group ID:

```
groups = [0, 0, 1, 1, 2, 2, …, 94, 94]
```

This ensures that if (T₁, T₂) is in the training fold, (T₂, T₁) is also in the training fold — **never in validation**. Without this, the model could "memorize" article-specific patterns from the original pair and trivially predict the swap, inflating CV scores.

## 9. Feature Selection

### 9.1 Variance Threshold

Features with variance below 10⁻⁶ are removed — these are near-constant features that contribute no discriminative information.

### 9.2 Mutual Information Selection

From the remaining features, we select the **top K = 250** by mutual information with the binary target label:

```
I(X; Y) = Σ_{x,y} P(x, y) · log[ P(x, y) / (P(x) · P(y)) ]
```

**Why mutual information?** Unlike Pearson correlation, mutual information captures both **linear and non-linear** dependencies between features and the target. Since our classifiers (XGBoost, LightGBM) make non-linear splits, MI-based selection retains features that are useful in practice even if they have weak linear correlation.

**Why K = 250?** With 190 training samples, having more than ~250 features risks overfitting (features > samples / 2). The MI selection acts as a dimensionality reduction step.

## 10. Model Architecture

### 10.1 Base Classifiers

We train four diverse gradient-boosted and tree-based classifiers, each hyperparameter-tuned via **Optuna** using Bayesian optimization (Tree-structured Parzen Estimator):

| Model | Key Hyperparameters | Optuna Trials | Why This Model |
|-------|-------------------|---------------|----------------|
| **XGBoost** | max_depth ∈ [2,4], gamma ∈ [0.5,5], lr ∈ [0.005,0.15], reg_alpha/lambda ∈ [0.1,20] | 150 | Strong single-model baseline, handles sparse features well |
| **LightGBM** | max_depth ∈ [2,4], num_leaves ∈ [7,31], min_child_samples ∈ [5,30] | 100 | Leaf-wise growth finds different splits than XGBoost |
| **CatBoost** | depth ∈ [2,5], l2_leaf_reg ∈ [0.1,20], subsample ∈ [0.5,0.9] | 80 | Ordered boosting prevents target leakage, handles small data well |
| **Random Forest** | n_estimators=500, max_depth=5, min_samples_leaf=3 | Fixed | Ensemble diversity — bagging vs. boosting |

**Regularization philosophy:** All tree models are heavily regularized:
- **Shallow depth** (max 3–5) to prevent memorizing individual samples
- **High L1/L2 regularization** (reg_alpha, reg_lambda up to 20)
- **Strong subsampling** (50–90% of rows and columns per tree)
- **Large minimum child weight** to prevent splits on few samples

### 10.2 Stacking Meta-Learner

The four base models produce **out-of-fold (OOF) probability predictions** via 5-fold StratifiedGroupKFold:

```
For each fold k:
    Train all 4 models on folds ≠ k
    Predict probabilities on fold k → stored in OOF matrix

OOF matrix: 190 × 4 (one column per base model)
```

A **Logistic Regression meta-learner** is trained on this OOF matrix:

```
P(label=2 | p) = σ(w₁·p_XGB + w₂·p_LGBM + w₃·p_CB + w₄·p_RF + b)
```

**Why LR for meta-learning?** With only 4 input features and 190 samples, LR is the least likely to overfit while still learning meaningful model combination weights. The weights effectively learn which base model to trust more for different probability ranges.

### 10.3 Rank Averaging (Alternative Meta-Learner)

As a non-parametric alternative:

```
P_rank(x) = (1/M) · Σ_m rank_m(x) / N
```

where rank_m(x) is the rank of sample x's probability among all N test samples under model m. Rank averaging is robust to probability miscalibration — it only cares about ordering, not absolute values.

We compare both meta-learners on OOF accuracy and use whichever performs better.

## 11. Pseudo-Labeling (Iterative Self-Training)

### 11.1 Motivation

With 95 training pairs and 1068 test pairs, the **unlabeled pool is 11× larger** than the labeled set. Pseudo-labeling exploits this asymmetry by using the model's own high-confidence predictions as additional training data.

### 11.2 Algorithm

```
Input: X_train (190 samples), y_train, X_test (1068 pairs)
Confidence thresholds: [0.88, 0.82, 0.78]

For round r = 1, 2, 3:
    1. Train full stacking ensemble on current training set
    2. Predict meta-learner probabilities on test set
    3. Select high-confidence test samples:
       - Positive pseudo-labels: meta_prob ≥ threshold[r]
       - Negative pseudo-labels: meta_prob ≤ (1 - threshold[r])
    4. For each pseudo-labeled sample:
       - Add original (t₁, t₂) with predicted label
       - Add swapped (t₂, t₁) with flipped label (augmentation)
       - Assign unique group ID (for GroupKFold integrity)
    5. Append to training set
    6. Retrain entire stacking ensemble on expanded data
```

### 11.3 Why Decreasing Thresholds

- **Round 1 (threshold=0.88):** Only the most confident predictions (top ~10-15% of test). These are almost certainly correct. Adds ~100-200 pseudo-labeled samples.
- **Round 2 (threshold=0.82):** Moderately confident predictions. The model has improved from Round 1, so more predictions are now reliable. Adds another ~100-200 samples.
- **Round 3 (threshold=0.78):** Lower confidence but still plausible. The cumulative training set may now be 400-600 samples, a 3-4× expansion.

### 11.4 Safeguards Against Error Propagation

| Safeguard | Implementation |
|-----------|---------------|
| High initial threshold | t=0.88 ensures only very confident predictions enter training |
| Minimum count | Skip round if fewer than 10 samples qualify |
| Balance monitoring | Track label balance (should stay ~50/50) |
| Group isolation | Pseudo-labeled pairs get unique group IDs |
| Fixed features | Test feature vectors are pre-computed and fixed; only training labels change |

---

## 12. Decision Threshold

Rather than using the default 0.5 probability threshold, we fix the decision threshold at **0.45**.

**Why?** The meta-learner probabilities can be miscalibrated on such a small training set. Optimizing the threshold on OOF predictions risks overfitting to the specific 190 samples. We observed that the auto-optimized threshold (often ~0.24 or ~0.72) performed well on OOF but generalized poorly to the test set, creating a large gap between public and private leaderboard scores.

Fixing at 0.45 trades a small amount of OOF accuracy for better generalization, reducing the public-private score gap.

## 14. Key Insights

1. **Perplexity is king:** Language model perplexity — especially the ratio between real-trained and fake-trained character LMs — was the single most discriminative feature class. Training domain-specific LMs from scratch on arXiv abstracts was a critical design choice.

2. **Hallucination ≠ paraphrase:** The manipulated model doesn't just rephrase poorly — it **fabricates entirely different content**. A real summary about VISTA telescope surveys might be paired with a fake about a "China relay network". This makes cross-pair entity overlap and content word Jaccard extremely powerful.

3. **Style breaks matter:** Manipulated text frequently breaks scientific register with exclamation marks, informal language, and vague quantifiers. These simple regex-based features are surprisingly discriminative and cost almost nothing to compute.

4. **LOO is essential:** Without Leave-One-Out perplexity computation, the language model features exhibit severe data leakage, inflating CV accuracy by ~3–5% while hurting test performance.

5. **Pseudo-labeling works:** The 1068 test pairs constitute 11× more data than the training set. Adding high-confidence predictions as pseudo-labels gave a meaningful boost, especially for rare edge cases not covered by the original 95 pairs.

6. **Threshold stability > accuracy:** The optimal OOF threshold was often far from 0.5, but overfits the tiny validation set. Fixing at 0.45 improved generalization and reduced the public-private leaderboard gap from ~2.5% to ~1.7%.

7. **Ensemble diversity matters:** XGBoost, LightGBM, CatBoost, and Random Forest each capture different patterns. The stacking meta-learner learned to weight them optimally, gaining ~1.5% over the single best model.

8. **GroupKFold prevents invisible leakage:** Without grouping swap-augmented pairs, a pair and its swap could land in different folds, artificially inflating CV scores by ~2% through article-level memorization.

## 16. Conclusion

We developed a hallucination detection system achieving **90.44% private / 92.12% public accuracy** on the "Catch the Liar" challenge, using only from-scratch representations. The key innovations were:

- **Domain-specific character and word language models** trained on 10,000 arXiv astrophysics abstracts, capturing the statistical fingerprint of genuine scientific writing
- **Hallucination-targeted features** exploiting the specific failure modes of manipulated LLMs (vagueness, informality, entity fabrication, topic drift)
- **Iterative pseudo-labeling** expanding the 95-pair training set using high-confidence predictions on the 1068-pair test set
- **Carefully regularized stacking ensemble** with GroupKFold validation preventing all forms of data leakage

The prohib­ition against pretrained models forced us to understand the problem at a fundamental level — engineering features that directly target the linguistic signatures of hallucination rather than relying on generic neural representations.
